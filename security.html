Architecture Notes

QR-Based Login Approval and Digital Access Verification System

Supabase Auth

User accounts and desktop sessions are managed with Supabase Auth. Protected pages verify the active authenticated session before loading identity records.

QR Session Verification

Each desktop request creates a temporary approval session. The QR contains only a public HTTPS approve.html link with a session ID.

Two-Device Approval

The desktop waits while the phone approves or rejects the session. The desktop uses 2-second polling to react to cloud database changes.

Access Logs

Approved, rejected, and expired outcomes are stored with UID, email, device, location, and timestamp for audit and thesis demonstration.

Cloud Database

Digital identities, approval sessions, and access logs are stored in Supabase tables with row-level security policies suitable for a public GitHub Pages frontend.

Session Expiration

Each approval request expires after 2 minutes. Expired sessions are marked in the database and rendered as a dark gray desktop state.

Role Concept

User roles remain attached to digital identities for future role-based access rules, such as student, employee, guest, and administrator access levels.

Prototype Scope

Inspired by NFC-style access systems, this prototype uses QR-based approval due to hardware limitations while preserving enterprise access-control behavior.
